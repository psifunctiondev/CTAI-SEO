# SEO Audit: Catherine Truman Architects
**Site:** catherinetrumanarchitects.com  
**Audit Date:** April 20, 2026  
**Prepared by:** Psi Function  
**Platform:** WordPress / Divi Builder  

---

## Executive Summary

The five highest-priority findings from this audit:

1. **🔴 Portfolio images have almost no alt text.** Across the 43-project portfolio — the core of the site — roughly 90% of images are missing alt text entirely. This is the single biggest missed SEO opportunity on the site. Image search drives significant traffic for architectural firms, and these images are effectively invisible to search engines.

2. **🔴 Most pages are missing an H1 heading.** The homepage, projects index, individual project pages, and the press page all lack an H1 tag in their HTML. H1s are how search engines understand what a page is about. The About page is the only exception. This is likely a Divi page builder configuration issue that can be corrected without a full redesign.

3. **🟡 robots.txt was broken but appears now fixed.** Prior analysis showed robots.txt returning a 404 error — meaning search engines couldn't confirm what they were allowed to crawl. The file now returns correctly with Yoast SEO generating it. However, **the fix should be verified in Google Search Console** to confirm Googlebot's view has updated.

4. **🟡 Server response time is slow.** Pages take 1.1–1.5 seconds before the first byte of content arrives. For a predominantly visual site where users bounce quickly, this is meaningful. Real-world load time (including images and JavaScript) will be significantly higher.

5. **🟡 No blog or original content.** The site has one placeholder blog post from September 2024. For a firm operating in a competitive local market (Boston architecture/interior design), the absence of any editorial content means the site cannot capture long-tail search queries — "how to renovate a historic Boston townhouse," "barn conversion architect New England," etc. — that are both high-intent and highly specific to CTAI's actual work.

### 1.3 Page Speed
**Status: 🟡 Needs improvement**

Server response times measured via curl (time to first byte + total download time):

| Page | Time to First Byte | Total Download | HTML Size |
|------|--------------------|----------------|-----------|
| Homepage | 1.25s | 1.33s | 111 KB |
| About Us | 1.51s | 1.63s | 205 KB |
| Projects | 1.17s | 1.26s | 128 KB |
| Project (Barn) | 1.16s | 1.26s | 178 KB |

**What this means for real users:** These measurements only capture the HTML document. Actual page load for users includes all images, stylesheets, JavaScript, and custom fonts — typically 3–8× the raw HTML load time. An architecture portfolio site is heavily image-dependent, so real-world load times are likely 5–15+ seconds without aggressive caching.

**Google's Core Web Vitals** (which directly affect search rankings) penalize slow sites. A Time to First Byte above 800ms is considered "needs improvement" by Google's standards.

**Contributing factors to investigate:**
- WordPress + Divi generates significant PHP processing overhead
- No evidence of aggressive server-side caching (Varnish, Redis, or similar)
- Multiple JavaScript libraries loading (jQuery, FancyBox, Owl Carousel, BeerSlider, custom scripts)

### 1.4 Mobile Responsiveness
**Status: ⚠️ Minor issue**

The viewport meta tag is set to:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=0">
```

The `user-scalable=0` setting **prevents users from zooming on mobile**. This is a known accessibility and usability concern. Google's mobile-friendliness guidelines flag this as a potential issue. It should be changed to allow zooming.

The Divi theme is otherwise responsive (media queries are present for tablet and mobile breakpoints).

## Section 2: Content & Structure

### 2.1 Heading Hierarchy (H1/H2/H3)
**Status: 🔴 Critical gap on most pages**

An H1 heading is the single most important on-page signal telling search engines what a page is about. Every key page should have exactly one.

| Page | H1 Present? | H2 Tags |
|------|-------------|---------|
| Homepage | ❌ None | None detected in initial HTML |
| Projects (portfolio) | ❌ None | "Projects" |
| Individual project pages | ❌ None | Project name (e.g., "Barn With a View") |
| Press | ❌ None | None detected |
| About Us | ✅ "Our Approach to Modernizing New England Home Design" | Team member names |

**The underlying issue:** The Divi page builder renders content primarily through JavaScript. When search engines (or our analysis tools) read the initial HTML, they see very little structured content. Google does execute JavaScript, but it happens in a second pass — meaning any content that exists only in the rendered DOM gets crawled less frequently and with less weight.

**What this means in practice:** A project page for "Barn With a View" — a custom home in Lincoln, MA — has its project name only as an H2 and no H1. This is a missed opportunity to signal relevance for searches like "custom home architect Lincoln Massachusetts" or "barn conversion architect New England."

### 2.2 Image Alt Text
**Status: 🔴 Critical — site-wide gap**

Alt text (alternative text for images) serves two purposes: it helps visually impaired users understand images, and it tells search engines what an image depicts. For an architecture firm, images are the product — they should be fully indexed.

**Findings across project pages:**

| Project Page | Images | Missing Alt Text | Coverage |
|--------------|--------|-----------------|----------|
| Barn With a View | 35 | 33 | 6% |
| Mountain Retreat | 18 | 16 | 11% |
| Cambridge Federal Renovation | 26 | 24 | 8% |
| Net Zero Barn | 24 | 22 | 8% |

The 2 images that DO have alt text on each page are the logo images ("CTA-logo_outlines") — not the portfolio photography.

**Across ~43 projects, this likely means 500–1,000+ images with no alt text.** Google Image Search is a meaningful discovery channel for architecture firms (potential clients searching "modern barn home New England"), and this is entirely untapped.

**The About Us page is an exception** — its 50 images all have alt text, suggesting the template used for that page handles it differently.

### 2.3 Blog / Content Section
**Status: 🔴 Essentially nonexistent**

The site has:
- 1 blog post: "Hello World" (placeholder, dated September 3, 2024)
- A "news" section in the sitemap, last updated April 2023
- A press section with 17 items (mostly from 2023, 3 recent from 2025-2026)

**Why this matters:** A blog is how a website captures long-tail search traffic — specific queries from people who are earlier in the decision-making process. Someone searching "how to add a mudroom to a Boston Victorian" or "net-zero home design New England" is a potential CTAI client. Without content targeting these searches, the site can only compete for brand-name queries ("Catherine Truman Architects") and a handful of high-competition generic terms.

Competitors who publish regular content on residential design, renovation trends, and Boston-area architecture will gradually outrank CTAI even for queries where CTAI's work would be the ideal answer.

### 2.4 Internal Linking Structure
**Status: 🟡 Limited visibility**

Because the site's navigation and content links are rendered by JavaScript (Divi builder), they are not visible in the initial HTML. This means our analysis cannot fully assess the internal link structure. However, from what is visible:

- The homepage HTML contains no visible navigation links in its initial source — all navigation is JS-rendered
- Projects contain breadcrumb navigation (good)
- No evidence of cross-linking between projects, between press coverage and related projects, or between About content and relevant projects

**Recommendation:** Add visible, HTML-native links between related content. For example: a press item about a barn renovation should link to the corresponding project page; a project page in Lincoln, MA should link to other New England projects. This both helps users and signals content relationships to search engines.

### 2.5 URL Structure
**Status: ✅ Clean and logical**

URL structure is clean, descriptive, and well-organized:
- Projects: `/project/[project-name]/` (e.g., `/project/barn-with-a-view/`)
- Press: `/press/[article-slug]/`
- Pages: `/about-us/`, `/projects/`

URLs are human-readable, use hyphens (not underscores), and describe the content clearly.

### 2.6 Social Media Presence
**Status: 🟡 Gaps in key platforms**

Social profiles linked from the site:
- ✅ LinkedIn (Catherine Truman personal profile)
- ✅ Instagram (@ctrumanarchitects)
- ❌ No Houzz profile linked
- ❌ No Pinterest profile linked

**Houzz** is the dominant platform for architect and interior designer discovery. Homeowners actively searching for professionals on Houzz represent high-intent leads. Not having a Houzz presence (or having one that's not integrated with the website) is a significant gap for a residential architecture firm.

**Pinterest** is notable: per the access log analysis, Pinterest drives more social referral traffic to the site than Facebook and Instagram combined. Yet there's no visible Pinterest integration or profile link on the site — the traffic is organic and unoptimized.

## Section 3: Search Engine Visibility

### 3.1 Indexing Status
**Status: Partially known**

Direct access to Google Search Console data was not available for this audit. The SearXNG search instance was limited (only Wikipedia available; Brave search suspended due to rate limiting), so automated ranking checks were not possible.

**What we can infer from the site's technical state:**
- The previous robots.txt 404 would have caused Google to be uncertain about crawl permissions. This may have created a crawl gap, particularly given the 516 Googlebot 404 errors in the server logs.
- The `x-robots-tag: noindex, follow` header was observed on the robots.txt URL itself (not a site-wide issue, just the robots.txt page)
- The site's robots meta tag is correctly set to `index, follow, max-image-preview:large, max-snippet:-1` on content pages — good

**Immediate action recommended:** Verify indexing status in Google Search Console. Specifically:
1. Are the 43 project pages indexed?
2. Are there coverage errors or excluded pages?
3. What was the crawl timeline around when robots.txt was broken?

### 3.2 Known Technical Crawl Issues (from Server Logs)
From the prior access log analysis:

| Issue | Details |
|-------|---------|
| robots.txt 404 | Previously blocking/confusing crawlers — now appears fixed |
| Total 404 errors | 48,649 (11% of all traffic) |
| Googlebot 404 encounters | 516 |
| YandexBot/Baiduspider | Fully blocked by rate limiting (429) |

The 48,649 total 404 errors at 11% of all traffic is unusual and warrants investigation. This could indicate:
- Broken internal links pointing to old URLs
- External sites linking to pages that no longer exist
- Changed URL structure without redirects
- Crawlers hitting paginated or parameterized URLs that don't exist

The 516 Googlebot 404s specifically should be investigated in Google Search Console's Coverage report to identify and redirect the most-impacted URLs.

## Section 4: Priority-Ranked Recommendations

### 🟢 Quick Wins (Low effort, high impact — do these first)

**QW-1: Add alt text to project images**  
*Why it matters:* This is the highest-ROI change on the site. Hundreds of professional photographs of CTAI's best work are invisible to search engines. Adding descriptive alt text (e.g., "Barn-inspired custom home in Lincoln Massachusetts with timber frame details") to project images would immediately improve visibility in Google Image Search, which drives discovery for architecture and interior design firms. This can be done in WordPress's Media Library or directly in Divi's image modules.

**QW-2: Add H1 tags to homepage, projects page, and project templates**  
*Why it matters:* Without an H1, search engines have a weaker signal about what each page is about. The homepage H1 should include the firm's primary keywords (Boston architecture, residential design). Each project page's H1 should be the project name — currently it's set as an H2. This is a Divi template edit, not a full redesign.

**QW-3: Verify robots.txt fix in Google Search Console**  
*Why it matters:* The robots.txt 404 is now resolved, but Google's crawlers may still have stale data from the period it was broken. Checking Search Console's Coverage report and submitting the sitemap for re-crawl will accelerate recovery. This takes 15 minutes.

**QW-4: Fix the mobile zoom restriction**  
*Why it matters:* Removing `user-scalable=0` from the viewport meta tag restores mobile accessibility and avoids a potential flag in Google's mobile usability checks. One-line code change.

**QW-5: Add Organization/LocalBusiness schema markup**  
*Why it matters:* Adding the firm's name, address, phone number, and social profiles in structured data (JSON-LD format) helps Google display a Knowledge Panel in search results and improves local search visibility. This can be added through Yoast SEO's settings without any coding. It takes about 20 minutes to configure.

**QW-6: Update the contact email to use the primary domain**  
*Why it matters:* The site displays `info@truman-architects.com` as the contact email, using the secondary domain. This creates brand confusion and misaligns with the primary domain. Should be updated to an `@catherinetrumanarchitects.com` address (or at minimum a consistent brand address).

### 🟡 Medium Effort (1–2 days of work, meaningful SEO improvement)

**ME-1: Investigate and resolve the 48,649 404 errors**  
*Why it matters:* One in nine requests to the server is hitting a broken URL. This wastes crawl budget (Google stops crawling a site that returns too many errors) and creates a poor user experience. Run a crawl report in Google Search Console or Screaming Frog to identify the top 404 URLs, then either redirect them to relevant pages or fix the broken links. Priority: fix the 516 Googlebot 404s first.

**ME-2: Create and optimize a Houzz profile**  
*Why it matters:* Houzz is the primary discovery platform for residential architects and interior designers. Homeowners actively searching for professionals use Houzz as a curated directory. A well-maintained Houzz profile with CTAI's portfolio photographs drives qualified leads directly. This is a significant gap for a residential firm.

**ME-3: Improve server response time (TTFB)**  
*Why it matters:* The 1.1–1.5 second time-to-first-byte is significantly above Google's recommended threshold (<800ms, ideally <200ms). This directly affects Core Web Vitals scores, which are a Google ranking factor. Options to investigate: enabling a WordPress caching plugin (WP Rocket, W3 Total Cache), server-level caching, or upgrading hosting. This requires technical WordPress expertise but the impact is site-wide.

**ME-4: Create location-specific project category pages**  
*Why it matters:* CTAI has projects in Boston, Cambridge, Brookline, Newton, Concord, Nantucket, Lincoln, and more. Adding crawlable category pages like "Boston Architecture Projects" or "Cambridge Renovation Projects" creates landing pages that can rank for location-specific searches. Currently, project categories exist in the sitemap but may not have optimized content.

**ME-5: Leverage Pinterest more deliberately**  
*Why it matters:* Pinterest is already CTAI's top social referral source — and it's completely unoptimized. Creating a verified Pinterest business account, organizing project boards (e.g., "Boston Residential Architecture," "Barn Conversions New England," "Modern Kitchen Design"), and adding Pinterest share buttons to project pages could significantly amplify this already-working channel.

**ME-6: Ensure JavaScript-rendered content is accessible**  
*Why it matters:* The site relies heavily on Divi's JavaScript rendering. Navigation links, many images, and content sections only appear after JavaScript executes. Google does crawl JavaScript, but it's slower and less reliable than server-rendered HTML. Working with a Divi developer to ensure critical content (navigation, H1s, primary CTAs) exists in the initial HTML would provide more consistent indexing.

### 🔵 Strategic (Longer-term, fundamental to competitive SEO)

**ST-1: Launch a content/blog strategy**  
*Why it matters:* This is the highest-impact long-term SEO investment available. The firm has 43 extraordinary projects, expertise in historic renovations, barn transformations, net-zero design, and New England residential architecture — and none of it is captured as searchable editorial content. A blog publishing 2–4 articles per month on topics CTAI clients actually search for would, within 6–12 months, build a substantial base of long-tail traffic. Topics could include:
  - "Renovating a Cambridge Triple-Decker: What We Learned" 
  - "Net-Zero Architecture in New England: Is It Right for Your Home?"
  - "Why Barn Conversions Are Having a Moment in Rural Massachusetts"
  - "5 Things to Know Before Renovating a Boston Townhouse"

**ST-2: Claim and optimize Google Business Profile**  
*Why it matters:* A verified Google Business Profile makes CTAI appear in Google Maps and the local "pack" (the map-and-listings block that appears for local service searches). For "Boston architect" or "residential architect Boston," appearing in the local pack is more prominent than standard organic results. This requires claiming the listing, verifying the address, adding photos, and maintaining it.

**ST-3: Build a structured internal linking strategy**  
*Why it matters:* The site has strong individual assets (press coverage, 43 projects, team profiles) but they don't link to each other in meaningful ways. A press mention of a project should link to that project page. A project in Cambridge should link to other Cambridge projects. The About page team members should link to projects they worked on. This "internal linking" signals to Google which pages are most important and helps users explore more of the site — both of which improve SEO.

**ST-4: Structured data for individual projects**  
*Why it matters:* Each project page could include `CreativeWork` or `ArchitecturalWork` schema markup identifying the project type, location, year, and architect. This is ahead-of-curve structured data that could generate rich results as Google expands how it surfaces design and architecture content. Lower priority than LocalBusiness schema but valuable long-term.
